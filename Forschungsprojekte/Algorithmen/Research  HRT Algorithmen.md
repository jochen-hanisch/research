## **1. Agentenbasierte Modellierung (ABM)**
## **Geeignet für: Komplexe Systeme mit interagierenden individuellen Agenten, deren Entscheidungen und Interaktionen das Systemverhalten beeinflussen.**
- Vorteile: Erlaubt die Modellierung heterogener Agenten mit eigenen Zielen und Entscheidungsprozessen. Sehr flexibel in der Darstellung individueller Unterschiede und Netzwerkdynamiken.
- Einsatzbereiche: Soziale Systeme, Marktdynamiken, Epidemienmodellierung, Verkehrssysteme.
## **2. Markov-Entscheidungsprozesse (MDP)**
## **Geeignet für: Entscheidungsfindung unter Unsicherheit, wo Zustände und Übergänge klar definiert werden können.**
- Vorteile: Gut für die Modellierung von Entscheidungsprozessen über die Zeit mit einer klaren, wohldefinierten Zustands- und Aktionsraumstruktur.
- Einsatzbereiche: Automatisierte Entscheidungssysteme, Robotik, Optimierungsprobleme.
## **3. Monte-Carlo-Simulation**
## **Geeignet für: Systeme, die stark von Zufallselementen beeinflusst werden und wo die Auswirkungen von Entscheidungen oder Zufallsereignissen quantifiziert werden müssen.**
- Vorteile: Nützlich, um die Auswirkungen von Unsicherheit und Zufälligkeit zu verstehen und Wahrscheinlichkeitsverteilungen von Ergebnissen zu generieren.
- Einsatzbereiche: Finanzrisikoanalyse, Projektmanagement, Kostenmodellierung.
## **4. Systemdynamik-Modellierung**
## **Geeignet für: Großskalige Systeme mit komplexen, rückgekoppelten Interaktionen und langfristigen dynamischen Veränderungen.**
- Vorteile: Erlaubt die Untersuchung von Feedbackschleifen, Verzögerungen und Akkumulationsprozessen. Gut geeignet, um das Verhalten ganzer Systeme über die Zeit zu verstehen.
- Einsatzbereiche: Umweltwissenschaften, Unternehmensstrategie, Public Policy.
## **Empfehlung für realitätsnahe Simulationen:**
- Agentenbasierte Modellierung (ABM) ist oft das vielseitigste und realitätsnächste Modell, besonders wenn das Verhalten und die Interaktionen von Individuen im Mittelpunkt stehen. Es ermöglicht die detaillierte Nachbildung von individuellen Verhaltensweisen und deren Auswirkungen auf das System, was es ideal macht, wenn individuelle Entscheidungen und deren Dynamiken im Zentrum der Analyse stehen.

import numpy as np

# Anzahl der Zustände
n_states = 21  # 20 Anweisungen + 1 Endzustand
n_actions = 3  # Ausführen, Überspringen, Test beenden

# Übergangswahrscheinlichkeiten
P = np.zeros((n_states, n_actions, n_states))
# Belohnungsfunktion
R = np.zeros((n_states, n_actions))

# Beispielfüllung der Matrizen P und R
for s in range(n_states - 1):
    P[s, 0, s + 1] = 1  # Aktion ausführen führt zum nächsten Zustand
    P[s, 1, s + 1] = 1  # Zum nächsten Zustand übergehen
    P[s, 2, n_states - 1] = 1  # Test beenden führt zum Endzustand
    R[s, 0] = 1  # Belohnung für das Ausführen einer Aktion
    R[s, 1] = -1  # Kosten für das Überspringen
    R[s, 2] = -10  # Hohe Kosten für das vorzeitige Beenden

# Implementierung der Value Iteration
V = np.zeros(n_states)
gamma = 0.95  # Diskontierungsfaktor
threshold = 1e-4  # Konvergenzschwelle
while True:
    V_prev = V.copy()
    for s in range(n_states):
        V[s] = max([sum(P[s, a, s_prime] * (R[s, a] + gamma * V_prev[s_prime])
                        for s_prime in range(n_states)) for a in range(n_actions)])
    if np.max(np.abs(V - V_prev)) < threshold:
        break

# Ausgabe der optimalen Politik
policy = np.zeros(n_states, dtype=int)
for s in range(n_states):
    policy[s] = np.argmax([sum(P[s, a, s_prime] * (R[s, a] + gamma * V[s_prime])
                               for s_prime in range(n_states)) for a in range(n_actions)])

print("Optimale Politik:", policy)

import networkx as nx

def draw_mdp_t1():

    G = nx.DiGraph()

    # Zustände und Aktionen hinzufügen

    for s in states[:-1]:

            G.add_edge(s, 1, action="Zyklus starten", weight=1)

## **Um einen Markov-Entscheidungsprozess (MDP) am Beispiel des medizinischen Algorithmus "Schmerzen (NRS ≥ 4) bei Trauma, Lumbalgie" durchzuführen, brauchen wir eine klare Definition der Zustände, Aktionen, Übergangswahrscheinlichkeiten und Belohnungen. ## **1. Definition der Zustände**
- Zustände (S): Diese könnten die verschiedenen Schmerzniveaus des Patienten repräsentieren (z.B. NRS 4-10), den Erfolg der Schmerzbehandlung (adäquat kontrolliert, nicht kontrolliert) und weitere medizinische Zustände, die aus dem Trauma resultieren.
## **2. Definition der Aktionen**
- Aktionen (A): Jede mögliche medizinische Intervention oder Behandlungsentscheidung, die getroffen werden kann, wie die Verabreichung verschiedener Schmerzmittel (Esketamin, Fentanyl), Anpassung der Medikation, Überweisung für weitere Diagnostik usw.
## **3. Übergangswahrscheinlichkeiten**
- Übergangswahrscheinlichkeiten (P): Wahrscheinlichkeiten, dass ein Patient von einem Zustand in einen anderen übergeht, basierend auf den durchgeführten Aktionen. Diese könnten aus klinischen Daten abgeleitet oder durch Expertenmeinungen geschätzt werden.
## **4. Belohnungen und Kosten**
- Belohnungen (R): Typischerweise in einem medizinischen Kontext als die Verbesserung des Patientenwohlbefindens oder die Minimierung von Schmerzen definiert. Kosten könnten negative Effekte, wie Nebenwirkungen der Medikation oder unzureichende Schmerzlinderung, beinhalten.
## **Modellierung des MDP**
## **Mit diesen Komponenten können wir ein einfaches Modell aufbauen. Zum Beispiel:**
- Zustände: {Schmerz NRS 4, Schmerz NRS 5, ..., Schmerz NRS 10, Schmerz kontrolliert}
- Aktionen: {Esketamin geben, Fentanyl geben, Behandlung anpassen, weiterführende Diagnostik veranlassen}
- Übergänge: Basierend auf der Effektivität und den Nebenwirkungen der Behandlungen.
- Belohnungen: Positive Punkte für Schmerzreduktion, negative Punkte für Nebenwirkungen oder unzureichende Kontrolle.
## **Durchführung des MDP**
## **Für die praktische Umsetzung des MDPs könnten wir programmatische Tools verwenden, um die optimale Politik zu berechnen. Diese Politik würde angeben, welche Aktion in jedem Zustand ausgeführt werden sollte, um die langfristige Belohnung (d.h. maximale Schmerzreduktion und minimale Nebenwirkungen) zu maximieren. Ein gängiges Verfahren zur Lösung von MDPs ist die Value Iteration, die iterativ die erwarteten Nutzenwerte für jede Aktion in jedem Zustand aktualisiert, bis die Werte konvergieren.**

import numpy as np

# Beispielhafte Definitionen (stark vereinfacht)
states = np.arange(7)  # NRS 4-10, Schmerz kontrolliert als Zustand 7
actions = np.array([0, 1, 2, 3])  # Esketamin, Fentanyl, Anpassen, Diagnostik
P = np.zeros((len(states), len(actions), len(states)))  # Übergangsmatrix
R = np.zeros((len(states), len(actions)))  # Belohnungsmatrix

# Beispielhafte Füllung von P und R mit fiktiven Daten

# Value Iteration
V = np.zeros(len(states))
gamma = 0.99
threshold = 0.01
while True:
    delta = 0
    for s in range(len(states)):
        v = V[s]
        V[s] = max(sum(P[s, a, s_prime] * (R[s, a] + gamma * V[s_prime]) for s_prime in range(len(states))) for a in actions)
        delta = max(delta, abs(v - V[s]))
    if delta < threshold:
        break

# Optimale Politik berechnen
policy = np.zeros(len(states), dtype=int)
for s in range(len(states)):
    policy[s] = np.argmax([sum(P[s, a, s_prime] * (R[s, a] + gamma * V[s_prime]) for s_prime in range(len(states))) for a in actions])

print("Optimale Politik:", policy)
